Detailed Description

该QItemEditorCreatorBase类提供​​了必须实施新的项目编辑器创造者时被继承的抽象基类。

QItemEditorCreatorBase对象是专门的窗口小部件的工厂提供的编辑器部件为一个特定的QVariant数据类型。它们被用来通过QItemEditorFactory创建编辑器QItemDelegate秒。造物主基地必须与注册QItemEditorFactory.registerEditor（ ） 。

编辑应为它编辑的数据提供了一个用户属性。然后QItemDelagates可以使用Qt的访问属性meta-object system设置和检索的编辑数据。一个属性被设置为与用户关键字的用户属性：

 Q_PROPERTY(QColor color READ color WRITE setColor USER true)

如果编辑器不提供用户属性，它必须从返回的属性的名称valuePropertyName（）;代表将使用此名称来访问属性。如果用户属性存在，项目代表将不会调用valuePropertyName（ ） 。

QStandardItemEditorCreator是可以用于注册的小部件，而不需要子类QItemEditorCreatorBase方便的模板类。

QByteArray QItemEditorCreatorBase.valuePropertyName (self)

这种方法是抽象的，应在任何子类中重新实现。

返回用于获取和创建者的编辑器窗口小部件设置值的属性的名称。

当实现在子类中此功能，您必须确保在这个函数中指定的编辑器部件的属性可以接受的创造者注册的类型。例如，制作者它构造QCheckBox小工具来编辑布尔值将返回checkable从这个函数的属性名称，并且必须在项目编辑器工厂登记QVariant.Bool类型。

注：由于Qt 4.2中的项目代表查询小工具的用户属性，只有调用这个函数，如果部件没有用户属性。您可以通过重新实现复盖此行为QAbstractItemDelegate.setModelData（）和QAbstractItemDelegate.setEditorData（ ） 。

See also QMetaObject.userProperty（）和QItemEditorFactory.registerEditor（ ） 。
